# Algorithm
알고리즘 연습

## 코딩테스트 어떻게 접근해야할까?
- 특정 문제를 특정 알고리즘으로 분류하려고 하지 말 것.
  - 알고리즘은 문제해결을 최적화하기 위한 방법일 뿐, 해당 방법이 최선인 것은 아니다.
- 설계와 구현을 별개로 생각할 것.
  - 설계를 끝낸 후, 문제를 구현해라.
  - 디버깅은 오타 및 실수를 잡기위한 것일 뿐, 잘못된 로직을 디버깅하기 위함이 아님.
- 문제에 대한 접근법
  - 완전탐색으로 해결할 수 있는 풀이를 먼저 생각할 것
  - 시간 복잡도를 계산할 것
  - 시간 제한에 맞춰 최적화할 것
    - 이 때 알고리즘이 사용된다.
- 쉬운문제부터 풀면서 구현 실력을 늘리고, 완전 탐색 문제를 많이 풀어 기초를 잡아야 한다.
  - 그 후에 알고리즘을 공부하기
